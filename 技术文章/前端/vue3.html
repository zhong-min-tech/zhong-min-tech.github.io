# vue3-emits

翻看Vue3官方文档，发现emits的一个新功能。官方文档是这样写的：

emits 可以是数组或对象，从组件触发自定义事件，emits 可以是简单的数组，也可以是对象，后者允许配置事件验证。

简单来说，就是emits可以定义一个验证函数，验证是否传递参数成功。

emits: {
    // 没有验证函数
    click: null,

    // 带有验证函数
    submit: payload => {
      if (payload.email && payload.password) {
        return true
      } else {
        console.warn(`Invalid submit event payload!`)
        return false
      }
    }
  }

这熟悉的飘红波浪……不慌，将update:modelValue用引号包上！

完美解决报错。

代码运行，跑起来~又是熟悉的小黄警告。

最后，验证函数写成这样就成功啦。

emits: {
    'update:modelValue': value => {
      if (value) {
        console.log(value)
        return true
      } else {
        console.warn(`Invalid submit event payload!`)
        return false
      }
    }
  }

# Vue 3.0 + Element plus 用递归组件实现动态菜单原理解析

动态生成多级菜单，是实际web开发中，百分之百会用到的一个必会技能。

接下来，就用Vue 3.0 + Element plus来实现下吧！

首先，我们拿到一个树状的数据。（实际开发中，通过请求后端获得）

[
  {
      "id": 1,     //菜单id
      "name": "菜单1",     //一级菜单
      "children":[]        //是否有子菜单，若没有，则为[]
    },
    {
      "id": 2,
      "name": "菜单2",
      "children":[]
    },
    {
      "id": 3,
      "name": "菜单3",
      "children": [
        {
          "id": 31,
          "name": "菜单3-1",//二级菜单
          "children":[]
        },
        {
          "id": 32,
          "name": "菜单3-2",
          "children":[]
        }
      ]
    }
]

希望在网页上呈现出这样的效果：

页面根据接收到的不同数据动态生成多级菜单。

即便菜单不断发生变动，也不需要修改代码。改动传入的菜单数据，就可以生成相应的菜单。不管这个菜单是一级菜单，还是二级菜单，甚至是三级菜单、四级菜单时，都可以快速、动态地生成。

如何实现呢？其实就是用到递归算法。

我们可以先用以下这段代码来理顺一下逻辑。

Menu.vue
<ul>
    <li v-for="item in data" :key="item.id">
        {{item.name}}
        <Menu v-if="item.children && item.children.length" :data="item.children"></Menu>
    </li>
</ul>

代码中data是前面提到的树状数据，循环树状菜单，获取到每一条数据。

v-if="item.children && item.children.length" 判断循环的那条数据里是否有子菜单。如果有就让组件调用自身，也就是Menu.vue这个组件，绑定子菜单数据:data="item.children"，继续循环生成菜单。

回到Vue 3.0 + Element plus，我们可以使用同样的思路，来动态生成动态菜单。

Element plus官网提供的演示样例是这样的：

<el-menu
    :default-active="activeIndex"
    class="el-menu-demo"
    mode="horizontal"
    @select="handleSelect"
  >
    <el-menu-item index="1">Processing Center</el-menu-item>
    <el-sub-menu index="2">
      <template #title>Workspace</template>
      <el-menu-item index="2-1">item one</el-menu-item>
      <el-menu-item index="2-2">item two</el-menu-item>
      <el-menu-item index="2-3">item three</el-menu-item>
      <el-sub-menu index="2-4">
        <template #title>item four</template>
        <el-menu-item index="2-4-1">item one</el-menu-item>
        <el-menu-item index="2-4-2">item two</el-menu-item>
        <el-menu-item index="2-4-3">item three</el-menu-item>
      </el-sub-menu>
    </el-sub-menu>
    <el-menu-item index="3" disabled>Info</el-menu-item>
    <el-menu-item index="4">Orders</el-menu-item>
  </el-menu>

可以将el-menu里面的内容单独封装成一个独立组件，如MenuItem.vue（可自定义命名）

相关代码就可以写成这样：

<el-menu-item v-if="item.children.length == 0" :index="item.id">
  {{ item.name }}
</el-menu-item>
<el-sub-menu :index="item.id" v-else>
  <template #title>{{ item.name }}</template>
  <template v-for="child in item.children">
    <menu-item v-if="item.children.length" :key="child.id" :item="child">
    </menu-item>
    <el-menu-item v-else :key="child.id" :index="child.id">
      {{ child.name }}
    </el-menu-item>
  </template>
</el-sub-menu>

menu-item是自定义组件MenuItem.vue。代码逻辑是先判断是否有子菜单，如果没有，生成一级菜单。有就调用自身，继续循环判断下一层。在下一层中，没有子菜单就生成菜单，有就继续循环判断。

这段代码用到的Element plus现成的样式，不过它实现递归动态菜单的基础逻辑与最上方的代码逻辑一致。

核心代码实现就是这些，至于其他细枝末节的内容这里就不补充啦。
