    # Langchain

    浅显比喻：Langchain之于大模型，就像是Spring至于Java。是统一封装的大模型，解决了不同模型不同API切换调用、不能连外网查询等问题，提供了一套更优雅的解决方案。

    开源地址：https://github.com/langchain-ai/langchain

    官网：https://www.langchain.com/langchain

    # LangSmith

    调试、测试、监控 LLM 应用的可观测性平台

    # LangServe

    将 LangChain 应用一键部署成 REST API 服务

    # LangGraph

    构建有状态、多步骤、可控循环的 LLM 工作流（基于图结构）

    # Langchain-RAG架构

    由于大模型token长度有限，每次和大模型对话，如果需要携带企业数据，可能超token，顺应诞生RAG架构。

    公开数据+企业私有知识库流程：读取解析私有知识，并对文档进行切割chunks,向量化后存储，解锁增强生成，prompt改写，通过大模型增强生成。

    # Langchain-Agent架构

    四个能力： 1、规划：复杂任务拆解、自我纠错 2、执行、 3、调用外部api、 4、记忆：长期记忆（如数据字典、指标字典、业务知识、云文档）、短期记忆

    # Langchain-六大模块

    # 1、Model I/O

    异步处理输入：ainvoke\abatch\astream

    # prompts

    1、动态插入提示词 2、few shot 模板 复杂问题做拆解后回答 余弦相似度、N-Gram匹配相似度，example-selector示例器构建模板 3、示例选择器

    # output parser

    # 2、Memory-内置api

    # 类别

    1、对话类 2、实体类 3、摘要类 存储信息，不会将历史信息全部存储，浪费token，一般会存最近几条信心 conversationEntityMemory:实体化记忆模块 conversationBufferWindow:记忆 conversationSummaryMemory:自动将冗长对话总结为精简摘要 k：number,保留多少条

    # 3、agent

    # chain只具备执行的能力，不具备规划能力，agent能执行更复杂的问题。

    # AgentExecutor：agent做决策，AgentExecutor执行

    提示词策略：reAct、self-ask with search、调用外部工具……

    ReAct:迭代循环直到得到答案，自我纠错 Self-Ask:通过搜索引擎解决问题 XML Agent:多次agent，中间数据采用xml格式 JSON Agent:中间传递json格式数据，提示词模板
